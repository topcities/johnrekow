Installing and Updating Windows 10 Without an Internet Connection


Windows 10 builds are always released as International Standardization Organisation (ISO) disk image files.
In Windows 10 Pro and Enterprise (not Windows 10 Home) you can double-click the ISO to "insert" the disk and view its contents.
You can also right-click an ISO and burn the disk to DVD, Dual Layer DVD (DVD-DL) or CD, depending on its size.
The Burn To Disk feature is not available in the Windows 10 Home edition.


The Windows 10 Installer can be downloaded for free from Microsoft.
You can download and use the installer for free, but it is the Product Key
that you will need to purchase and activate.

Most newer Windows devices have a Windows product key already memorised in the device's hardware.

If you are building a machine and you do not want to connect it to the internet until the Windows Updates are installed first, there is a way to get and install all of the Windows Updates up through last month without an Internet connection.


Windows Updates are also available in an all-inclusive installer.
The installer will install all major updates up through the month indicated.

More precisely, the Cumulative Update app has all of the updates up through the third week of the month it was released.
Updates that are more than three weeks into the current month will be included on next month's Cumulative Update app.

Each month, a new Cumulative Update app is created for Windows 10.
The title has the format of YYYY-MM each month.

Do a search using syntax as shown below.


Microsoft prefers to mention the year first, followed by the month.

The syntax is
Windows (build number YYMM) (year) (hyphen) (month) and the words, Cumulative Update

Example
Windows 1703 YYYY-MM Cumulative Update

To do a search for the Cumulative Update for August 2017:
Windows 1703 2017-08 Cumulative Update

That would have all updates up through August 2017, for Windows 10 Build 1703, which is the build that was released in 2017, March.

If you did a search using the above line,
you would find the installer that is the Cumulative Update for Windows 10, Build 1703 (build released in 2017 March) Updater that has all updates through 2017 August (2017-08).

If you want to know what build or version you have, navigate to this About Windows app and launch it:

C:\Windows\System32\winver.exe

The version number is the build number (not the OS build number shown inside the round brackets).

Windows 10 System Requirements




Windows 10 System Requirements


Generally, a machine that is running Windows 7 SP1 or Windows 8.1 can also run Windows 10.
There are also some machines that ran Windows XP that could also qualify for running Windows 10.

Windows 7 SP1 or Windows 8.1 can be upgraded without having to erase and install Windows 10 from scratch.

Windows 10 System Requirements
Processor:
1 gigahertz (GHz) or faster processor or System on Chip (SoC)

RAM:
1 gigabyte (GB) for 32-bit or 2 GB for 64-bit

Minimum Hard Disk Space:
16 GB for 32-bit OS, or 20 GB for 64-bit OS

Graphics card:
DirectX 9 or later with Windows Display Driver Model (WDDM) 1.0 driver

Display:
800x600

Personal advice:  As of 2017, 1 single processor will work, but very slowly. Avoid using a machine with only one single processor.
2 processors are tolerable.  4 processors would be acceptable.
1 gigahertz (GHz) processor speed is a bit slow, but acceptable.  Go after a faster speed.  Above 2 GHz is good, and above 3 GHz is very good.
Most machines are 64-bit.  Avoid 32-bit machines, since some software is no longer made for 32-bit.
There used to be some 64-bit machines that only ran 32-bit software, because the hardware drivers were only released in 32-bit.  Mind out for those devices, and avoid them.
For 64-bit, 2 GB of RAM will work, but 4 GB or more is better.
For Hard Drives, 20 GB is the minimum for 64-bit Windows 10.
However, based on the amount of typical apps that one would install, and the typically-expected amount of personal file accumulation that could be found on a hard drive,
it is personally recommended that the hard drive be no less than 128 GB.
128 Gigabytes (GB) = 131072 Megabytes (MB)
If you are going to buy a Windows device, and there is not at least 128 GB of hard disk capacity, I would advise you not to purchase it.


There are many additional notes and details about the system requirements and the upgrade process.
To find out about those details, visit the page below:

Windows Boot to CD, DVD, USB




Windows Boot to CD, DVD, USB


Booting Up to a CD or DVD in Windows 8.1 OEM Builds -- Disable Secure Boot

The steps and procedures very, depending on the manufacturer.
Usually there is a certain Function Key to press that will change the boot order or choose the optical drive or CD/DVD drive as the device to boot from.

Another way is to go into the Basic Input/Output System (BIOS).  Note that the BIOS technology is changing and is being replaced with new technology, like, for example, Unified Extensible Firmware Interface (UEFI) which supports Secure Boot, so these instructions may already be obsolete for new machines, as BIOS fades away and is replaced with UEFI.
The way to get into the BIOS varies.  It also usually involves repeatedly pressing a certain Function Key (F-Key, F1 through F9 on the keyboard.)  Some keys typically used to boot into the BIOS are F1, F2, F12, or Escape (Esc).
Once at the BIOS menu, usually, you use only the arrow keys to navigate, the Enter key to select or make changes, and the Escape (Esc) key go back out or Cancel.
In most cases, navigate to System Configuration (or something with a similar name) and look for Boot Options, where you can control which devices are allowed to boot the machine, and in which sequence they will be examined for possible booting software.
Make sure CD-ROM Boot, or CD/DVD boot, or sometihng to that effect, is Enabled.  If there is an order that the devices are in, change the order so that the optical drive is examined first.  Look across the bottom of the BIOS screen for instructions on how to select and move devices up and down the list.  The machine will start at the top of the list and look at that device to see if it has boot software.  If the drive is empty or does not contain bootable software, like some sort of bootable operating system, it will go to the next device in the list and see if it has an operating system that it can boot from. 

While in this area, look for an option called "Secure Boot."  If you are not able to get your bootable DVD or bootable CD to run and boot up, you may need to Disable Secure Boot.  Only newer machines have this option, and where you can enable and disable Secure Boot in the BIOS will vary.  If you do need to disable Secure Boot, you may also need to tell Windows that it is disabled as well.
Try to see if you can boot from DVD or CD while keeping Secure Boot on. If bootup by CD/DVD still does not work, then go into BIOS and Disable Secure Boot.
There also may be circumstances where you need to disable Secure Boot in order to install a graphics card, some hardware, or other operating system that’s not compatible with Secure Boot.

Secure Boot helps to make sure that your PC boots using only firmware that is trusted by the manufacturer.  Secure Boot is a security standard developed by members of the PC industry to help make sure that your PC boots using only software that is trusted by the PC manufacturer.
When the PC starts, the firmware checks the signature of each piece of boot software, including firmware drivers (Option ROMs) and the operating system. If the signatures are good, the PC boots, and the firmware gives control to the operating system.
For most PCs, you can disable Secure Boot through the PC's firmware (BIOS) menus, as mentioned previously. For logo-certified Windows RT 8.1 and Windows RT PCs, Secure Boot is required to be configured so that it cannot be disabled.


Another way to Disable Secure Boot is to do it from Windows, rather from inside the BIOS.
After windows starts up and is running, hold the Shift key while selecting Restart. Go to Troubleshoot > Advanced Options: UEFI Firmware Settings.
Find the Secure Boot setting, and if possible, set it to Disabled. This option is usually in either the Security tab, the Boot tab, or the Authentication tab.
Save changes and exit. The PC reboots.


If you see a "SecureBoot isn't configured correctly" watermark in the lower-right corner of the desktop, a Windows Update has been released that will fix that and remove the watermark. In that case, run the Windows Update Control Panel.

Many technicians are also taking their bootable software and installing it on a USB stick (thumb drive), so that they can boot from a USB stick.  It is possible to boot up the Windows Installer and install Windows with a USB stick, instead of a DVD.  Many new tablets and devices do not come with an optical drive, so the bootable USB stick is a popular way to install Windows.

The instructions above on how to go about booting from an optical drive also should work if you want to make a USB stick (flash drive or thumb drive) be a bootable device.  Just select the option for USB instead of the option for an optical drive in the instructions above.  The remaining information applies only to those running Windows RT.

Booting up a Windows RT device using an optical drive or USB has been made to be intentionally impossible for regular users, yet some technicians are starting to report that there may be some situations where Booting by USB stick and turning off Secure Boot may be possible in Windows RT in some rare situations, although it is intentionally supposed to be impossible to do.

There may be three possible ways to do this:
Volume Down + Power Button  or
Volume Up + Power Button  or
Volume Up + Windows Key on Tablet

Again, one of the three possible ways to boot up a machine to a USB stick, running Windows RT, on some machines, is to hold down the volume down while powering on.

On other models, you hold "Volume Up" and the Windows key on the tablet itself (not the keyboard Windows key).

A third way may be to press and hold both the hardware Volume Up button (usually on the left side of the RT device) and press the Power button (usually on the top). Then, release the Volume Up button.

Another way from within Windows, is to go to the Charms Bar on the right, Settings, Change PC Settings, and then navigate to Update and Recovery, then Recovery. Then, tap the Restart button under Advanced Startup, and when the system reboots into the Recovery environment, choose Troubleshoot, Advanced Options, and then UEFI Firmware Settings. Then, tap Restart.

Some of the options you may find in the UEFI firmware interface:

Delete All Secure Boot Keys. If you select this option and then Yes, Secure Boot will be changed from the usual User mode to Setup mode. In this mode, Secure Boot is disabled and the option changes to Install All Factory Default Keys. Select that and then choose between "Windows & 3rd party UEFI CA [certificate authorities] (Default)" and "Windows only."

Configure External Ports. If you choose this option, you'll be presented with a pop-up menu with the following choices: All ports enabled (the default), Enable USB & microSD/Disable Docking Port, Enable Docking Port/Disable USB & microSD, and All ports disabled.

Device Information. If you choose this option, you'll be presented with a pop-up that displays your Surface Pro 3's System UUID (universally unique identifier) and serial number, both of which are used to uniquely identify your device.

About every 6 months, a newer build of Windows 10 is released and sent automatically to all Windows 10 machines using Windows Update.
Some of these instructions were based on older builds.
In newer builds, registry keys that did not exist a year or two ago may now already exist.
If the steps below ask you to create a key that already exists, it is because the newer Windows 10 builds now include that key by default.
It is good to observe the instructions below and at least check the key names, locations, and the value (1 or 0) being stored in the key, even if they now exist already by default.
Also, the exact wording, names and locations of preferences and settings may change in newer builds.
If these instructions don't quite match what you see in your newer build, you will have to figure out what the new name is, and where it may have been moved to.
Some settings may also be adjusted by default in newer builds.
Some items that used to be turned off by default may now be turned on by default, and what would have previously been on in an older build may now be found to be off by default in the newer build.

The build number refers to the year and month when that version of the software was finished.

The finished build is usually not released until the following month.

Prohibit Certain Apps From Running


Make any app stop running and not be allowed to run...

In the Registry Editor, navigate to the following key:
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies

Make more keys, more of the yellow folders
inside Policies
to be structured like this...
Policies
--Explorer
----DisallowRun


Make a new key (a new yellow folder that will be inside)
right-click on Policies, and then select New, Key,
give the new key the name of, Explorer

make a new key (a new yellow folder that will be inside)
Right-click on the Explorer key, and then select New, Key,
give the new key the name of, DisallowRun

The apps listed inside DisallowRun will all be listed as values
in the right column.

Select the DisallowRun key on the left pane.
then, over in the right column
Right-click on an empty area and select New, String Value.

Make a list like this
Name Data
(Default) (value not set)
1 appname1.exe
2 appname2.exe
3 thirdappname.exe
4 anotherapp.exe

Name the first new entry with the number 1
and set its value to the name of the application(.exe)
that you want to block.
If you want to restrict more apps,
simply create more String entries with names 2, 3 and so on
and set their values to the app's exe file name.

Restart your computer and try to run the blocked applications
and you'll get the error message,
"This operation has been cancelled due to restrictions
in effect on this computer. Please contact your system administrator."

This is dependent on the name of the app not being changed.
If you rename the blocked .exe app, it will run.








.NET Installation for Windows 10

Here is how to install .NET onto Windows 10 without using an Internet connection.

You will need the Windows 10 Installation DVD or the ISO file thereof.

Insert the Windows 10 DVD or double-click the DVD ISO file and note the drive letter assigned to that DVD drive.

Launch Command Prompt (right-click to Run As Administrator) and execute this command:
You must change the drive letter shown below (e:) so that it matches the drive letter assigned to the DVD.
dism.exe /online /enable-feature /featurename:NetFX3 /Source:e:\sources\sxs

There is a space before each forward-slash, but not before each back-slash.

The Command Prompt will let you know when the installation is completed.

After the operation completes, go ahead and "eject" the physical or virtual DVD.

Restart the Computer, and .NET will be installed on Windows 10.

Editing  the  Boot-Up  Descriptions

This is how to change the boot descriptions on a multi-boot machine, where you have to choose an operating system during startup.

On a multi-boot machine, if the description of each boot option is unclear, here is how to change it:

1. Start your multi-boot computer, and when the choices of operating systems appear, choose one of the boot options from the boot menu.

The boot option that you select during startup will be the one that will have its boot-up description edited.

After startup completes, make a note of which Windows installation is running.


2. Run As Administrator the Command Prompt (cmd.exe)

3. Type the following command: bcdedit /set description "Menu description goes here"
Put inside the quotes your own description for the boot description text, and be sure to include the quotation marks in your command. Press Enter.

4. Restart your computer, and note that the menu description you just entered now appears on the menu. Select the other boot menu option.

5. Repeat steps 2 and 3, again adding a menu description to replace the generic text in order to make the boot description more unique and meaningful.
