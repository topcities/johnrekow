Automatic Backups




Automatic Backups

rsync helps to make automatic backups of your data, so that, if your local machine should die, you can still find all of your files safely copied into another location elsewhere.

rsync is fast, incremental file transfer Freeware for Linux.

rsync is good for copying large amounts of files quickly, but rsync is only good for a small number of servers.

rsync has to be installed on both the source and destination machines.

rsync is already installed on the CentOS minimal build.

rsync -avzh
In this flag;
-a is archive
-v is verbose
-z is compress ("zip")
-h preserve hard line
-x do not cross file systems

It will compare the file lists and the last-modified dates & times, to determined which newly-changed files need to be backed-up.

Automatic HTTP to HTTPS




Automatically Redirect Users From HTTP to HTTPS


LXOS:  Apache Software Foundation Top-Level Wiki:  Force HTTP to HTTPS -- Force Browsers To Only Use HTTPS When Visiting Your Site
http://wiki.apache.org/httpd/RewriteHTTPToHTTPS

The code to use is below.  The hash character ( # ) indicates that this line contains a comment and is to be ignored.

# RewriteEngine On -- will enable the Rewrite capabilities

RewriteCond %{HTTPS} !=on
# Checks to make sure the connection is not already HTTPS

RewriteRule ^/?(.*) https://%{SERVER_NAME}/$1 [R,L]
# This rule will redirect users from their original location, to the same location but using HTTPS.
# i.e. http://www.example.com/foo/ to https://www.example.com/foo/
# The leading slash is made optional so that this will work either in httpd.conf or .htaccess context

Put the above solution in a .htaccess file in the directory to be affected.

Another Solution:

The following code should work with .htaccess

# Code:
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteBase /
RewriteCond %{HTTP_HOST} !^domainname.tld$
RewriteRule ^(.*)$ https://domainname.tld/$1 [R=301]
</IfModule>








BASH Commands




Miscellaneous BASH Commands



This BASH command finds a programme or file by searching the user's path:
which
This will search for the bash directory:
which bash

Search all installed repositories
yum search

List everything installed
yum info

Create a new, empty file
touch nameOfNewEmptyFile


Change file permissions
chmod 777 myFileName

The three digit number refers to a how much freedom and permission a user has.
The numbers range from zero to seven.
Seven gives complete freedom to read, write and execute. Zero will prohibit all of those actions.


The first digit on the left is for the owner, the middle digit is for the group, and the third digit on the right is for "other."
The numbers result in different combinations of permission.
For example, 4 will allow one to only read a file, but not execute it if it is a script or application.

The following website has an online cmod Permissions Generator.
You can tick the permissions that you want the owner, group and other to have,
and the generator will give you the corresponding three-digit number, based on your selections.

Collecting Static Network Connection Details




Collecting Static Network Connection Details


Collecting Static Network Connection Details on a Linux machine.

See Also the Ifconfig section on this page.

Normally on a PC, you launch the Command Prompt (right-click and Run As Administrator) and type
ipconfig /all
and that shows you the connection details on a PC running Windows.


On a Linux machine running Xfce, you click on the Applications Menu, in the top left corner of the screen,
Navigate to here:
Accessories, Terminal Emulator

The name and location of the Terminal may vary on different versions of Linux.
For example, on some versions of the Fedora build of Linux, the terminal is called, "LXTerminal" and is easily found as a black icon in the bottom left corner of the screen, or it may be found listed under System Tools.

After the terminal is up and running, on Ubuntu, type the following with the spaces as shown:
ifconfig -a

If the above command does not work or is "not found", see the ifconfig section on this page for other commands that might work.

The default gateway is called the "Bcast" or Broadcast.

The only thing not shown is the DNS server.
To get the DNS configuration, type and enter the following:
cat /etc/resolv.conf

There is a space after cat.

If that did not work, try removing the space after cat, like this:
cat/etc/resolv.conf




The above information usually works. However, if you are still not connecting, the current network information may be stored in a different location, using a different procedure that points to a different network script file.

Try executing the following command:
cat /etc/sysconfig/network-scripts/ifcfg-eth0

There is a space after cat. That last character is the number zero.

This will display all of the information you need to make a static connection.
If this is being used instead, you may see some values here that do not match what you collected earlier.

Compare these results with the information collected earlier, and try using any different values displayed here.





Here is another place that can have Connection Information.

If the connection information is not found in the typical locations above,
execute this command:

/sbin/ifconfig/

This is not a directory.
This is not a text file that you can open with cat, more or vi.

Execute the command exactly as it is shown to display the details.

http://www.cyberciti.biz/faq/how-to-find-out-the-ip-address-assigned-to-eth0-and-display-ip-only/

Disable Apache ServerSignature




Disable Apache ServerSignature

Disable Apache ServerSignature Commands After Launching New Blank CentOS Minimal



You may need to navigate to the Root Directory before going to the files and directories mentioned in this section.

Change Directory to Root Directory:
cd /

Change Directory to Home Directory:
cd ~

By default, the apache logs are stored in the directory:

/var/log/httpd/



How to disable apache server signature on your web pages:
To disable server signature, you will need to edit your /etc/apache2/apache2.conf file.
vi /etc/apache2/apache2.conf

If the above cannot be found from the Root Directory, then locate the following instead.

/etc/httpd/conf


main apache config file -- extremely important!
/etc/httpd/conf

Edit this file found in the above directory (this command opens the file in vi):

vi httpd.conf

Again, that file is located here:
/etc/httpd/conf/httpd.conf


In vi, to search for this text, place a forward-slash in front of your search term, like this:
/ServerSignature


Make the following edits:

ServerSignature Off
ServerTokens Prod

If that text cannot be found, just scroll to the end of the file and add those two lines of text.
Then, we need to reload apache configuration to put this new change in effect:

sudo /etc/init.d/apache2 reload


The above instructions are well-documented. Here is an example of one source:

If the ServerTokens is set to Min

Ifconfig




Ifconfig Commands

Network configuration and status for Linux OS

ifconfig – status of all network interfaces
ifconfig eth0, – status of ethernet 0 connection
ifconfig eth0 down – shuts ethernet 0 down
ifconfig eth0 up – starts ethernet 0
ifconfig eth0 100.222.255.255 – assigns that IP address to ethernet 0
man ifconfig, – more info

See also the section on this page, Collecting Static Network Connection Details on a Linux machine.

If ifconfig does not work or is "not found,"
try this command instead:
/sbin/ifconfig

The above command is necessary when ifconfig is placed in the
/usr/sbin directory
which is not in a non-admin user's path.

This has been the case in some Linux releases.
For example, in the 2014 releases of the Debian OS,
the only way to access ifconfig was to type the above command, which is, again:
/sbin/ifconfig

MySQL on Linux




MySQL on Linux

MySQL General Installation Guidelines

These are not the precise steps.
This is just a general reminder of the main things to do in order to Install MySQL on a Linux machine.

Install package
Edit the config file /etc/my.cnf
Start the service
MySQL binds to a port, like everything else
Can bind to a UNIX socket
Run the secure installation script

 


To reset the MySQL Root password.

1. Stop MySQL (assuming this a CentOS machine)
sudo /etc/init.d/mysqld stop

2.Restart in Safe Mode
sudo mysqld_safe --skip-grant-tables &

3. Login
mysql -u root

4. Specify the Database to use
use mysql;

5. Reset Password
The following command enter the new password:
update user set password=PASSWORD("mynewpassword") where User='root';

6. Flush privileges & Quit
flush privileges;
quit

7. Stop MySQL
sudo /etc/init.d/mysqld stop

8. Wait Until Stopped, Then Start it up
Wait until the stop has completed, the start it up again.
sudo /etc/init.d/mysql start

9. Login With New Password
mysql -u root -p

These steps came from the Rackspace Knowledge Center, and were publish in August 2012 at the following URL:
http://www.rackspace.com/knowledge_center/article/mysql-resetting-a-lost-mysql-root-password

Powering on CentOS




Powering on CentOS

On a Windows machine, CentOS virtual boxes do not have mouse abilities and they do not copy and paste.

To get the mouse, copy and paste to work on a CentOS Virtual machine, login to that machine
using the Windows app, PuTTY.

Here are the steps on how to do this.


Turn on the Ethernet virtual network card to enable network connectivity.
In order to turn it on, you need to give the Ethernet a name.
In this example, we give the Ethernet the name of "eth" followed by the number zero:
ifup eth0

The Ethernet must be enabled after each and every boot or restart.


You only need to do this one time to make the Secure Shell (ssh) enabled.
In the CentOS virtual machine, powered by Oracle Virtual Machine VirtualBox app,
go to the Machine menu, select Settings, on the left, choose Network.
Expand the Advanced section.
Make sure Cable Connected is ticked.
Click the Port Forwarding button.
Click the green plus-symbol icon to add a new Port Forwarding Rule.

Here is an example of the details to enter.
You can choose whatever port numbers you want.
Name: SSH
Protocol: TCP
Host IP: 127.0.0.1
Host Port: 2222
Guest IP: (leave this field blank)
Guest Port: 22

Click OK.

Start the Virtual Machine.
login.

after login, execute this command:
ifup eth0

then you can login using Putty on the same machine.


to connect to ssh:

Enter the Host Name and Port number:
The host name may vary. One commonly used username is root.
username@localhost
Port 2222
Connection Type SSH


To end your ssh session in Putty, execute this command:
exit


Other power-related commands:
Restart:
reboot

Shut Down:
poweroff

ShutDown




Shut Down


Shut Down Linux

On a GUI like Debian, in the top right corner, select Log Out. Click Log Out, then go to the top right corner again and select Shut Down or Shutdown.
On most GUIs, the Shutdown option usually does not appear until after you log out first.

If you are not in a GUI, here is how to shutdown a Ubuntu Server
Linux command to shutdown server / computer immediately:
$ sudo shutdown -h now
or...
$ sudo shutdown -h 0
-h stands for halt.

-r stands for reboot:
$ sudo shutdown -r 0

On a server machine, you may be required to enter a password to have the command executed.

TAR Commands




TAR Commands
tar combines folders and files together, but does not compress them.

tar commands
- Glue files together:
tar -cf tarfile file List
- Verbose output:
tar -cvf tar File file List
- List files in a tarFile:
tar -tf tar File
- Extract files from tarFile:
tar -xvf tarFile
"xvf" means to eXtract, then Verify the File with that tarFile name.
Note: The hyphen is not needed in the above commands.
(tar was invented before the use of the hyphens)

Zabbix Basic Installation Guide

This is a general guide only. The precise steps and commands are not mentioned here.

You do these thnigs inside the virtual machine, then you build it in openStack.

Some steps have not been verified as all being entirely accurate.

1. Enable epel

2.Install MySQL Server

3.Create Zabbix Database & User

4. Install Zabbix\20 Server
Do a yum search to help find things to complete this step.

5. Import
usr/share/zabbix-mysql/
Xabbix will make this for you: schema.sql

6. SELinux: You may diable SELinux.
If you do not turn it off, then you will need to poke holes in it so that the database can connect.

7. Go to your webpage.../zabbix
Install the script. Point it to the database. Click Next.
It will do the installation for you.

8. Edit this configuration file
/etc/zabbix/zabbix_server.conf

9.Install zabbix20-agent on web server.
Also, edit the monitoring parameters in the configuration file located here:
/etc/zabbix20.agentd.conf

10. Add the host in the Zabbix interface.

The easiest way to get Zabbix to watch something is to install the Zabbix agent on that something.
to do that, use this command:
yum install zabbix20-agent

After you login to the Zabbix webpage, click on the Configuration tab,
and give it the IP address that you want to monitor.

To look at these, click the Monitoring tab.
